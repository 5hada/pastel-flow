# Pastel Flow 세부 UI 설계

이 문서는 `ui-features.md`의 Action/Workflow 기반 기능 범위를 화면 구조, 상태, 컴포넌트 규칙으로 구체화한다. 기본 목표는 사용자가 Workflow를 빠르게 실행하고, Action을 조합해 새 Workflow를 안정적으로 작성할 수 있는 작업 앱 UI를 만드는 것이다.

## 1. 전체 앱 프레임

Pastel Flow는 창 전체를 쓰는 작업 앱 프레임이다. 앱은 상단 카테고리 바, 좌측 탐색 패널, 우측 작업 영역으로 고정 구성한다.

### 구성

- 상단 카테고리 바: 앱 이름, 현재 모드, 실행, Action, Workflow, 도구, 설정, 새로고침
- 좌측 탐색 패널: 현재 모드의 필터, 목록, 카테고리, 설정 분류
- 우측 작업 영역: 런처 목록, 상세, 작성 폼, 도구 실행, 설정 세부 화면

### 배치 규칙

- 데스크톱에서는 상단 바 아래 좌측/우측 2열을 유지한다.
- 좌측 패널은 화면 높이를 채우며 내부 스크롤만 가진다.
- 우측 작업 영역은 목록이나 폼이 길어질 때 독립적으로 스크롤한다.
- 전체 창 자체에는 별도의 가로 또는 세로 스크롤바를 만들지 않는다.
- 스크롤은 좌측 패널의 세로 스크롤과 우측 작업 영역의 세로 스크롤만 허용한다.
- 전체 화면을 감싼 큰 카드 레이아웃을 만들지 않는다.
- 화면 폭이 좁으면 좌측 패널을 접을 수 있게 한다.
- 좌측 패널 토글 버튼은 패널 내부가 아니라 앱 프레임의 좌측 끝 고정 위치에 둔다.
- 좌측 패널이 열리거나 닫혀도 토글 버튼의 화면상 위치는 급변하지 않아야 한다.

## 2. 상단 카테고리 바

상단 카테고리 바는 앱의 주요 모드를 전환한다.

### 구성

- 좌측: `Pastel Flow`, 현재 모드 라벨
- 중앙: 실행, Action, Workflow, 도구, 설정
- 우측: 새로고침

### 액션

- `실행`: Workflow 런처 화면으로 이동한다.
- `Action`: Action 목록과 편집 화면으로 이동한다.
- `Workflow`: Workflow 목록과 편집 화면으로 이동한다.
- `도구`: Tool Module 루트 폴더 등록, 검증, 관리, 단독 실행 화면으로 이동한다.
- `설정`: 앱 전역 설정 화면으로 이동한다.
- `새로고침`: 현재 모드에 필요한 로컬 데이터를 다시 불러온다.

### 상태 표시

- 로딩 중에는 새로고침 버튼을 비활성화한다.
- 공통 오류가 있으면 상단 바 아래 오류 배너에 표시한다.
- 현재 기기 신뢰 상태는 우측에 짧은 배지로 표시할 수 있다.

## 3. 좌측 탐색 패널

좌측 패널은 현재 모드에 따라 내용이 바뀐다.

### 실행 모드

- 전체
- 실행 중
- 예약됨
- 실패
- 제한됨
- secret 필요

### Action 모드

- 좌측 패널에는 Action 목록을 표시한다.
- 목록 우상단에는 새 Action 생성을 위한 `+` 아이콘 버튼을 둔다.
- 목록은 최종적으로 폴더 또는 그룹 기반 계층 구조를 지원할 수 있게 설계한다.
- 초기 평면 목록에서도 `전체 Action`, `브라우저`, `Crawler`, `Dry-run`, `Tool Action`, `사용되지 않음` 필터를 제공할 수 있다.

### Workflow 모드

- 좌측 패널에는 Workflow 목록을 표시한다.
- 목록 우상단에는 새 Workflow 생성을 위한 `+` 아이콘 버튼을 둔다.
- 목록은 최종적으로 폴더 또는 그룹 기반 계층 구조를 지원할 수 있게 설계한다.
- 초기 평면 목록에서도 `전체 Workflow`, `최근 수정`, `예약됨`, `제한됨`, `실패` 필터를 제공할 수 있다.

### 도구 모드

- 좌측 패널에는 등록된 Tool Module 목록을 표시한다.
- Tool Module 목록은 좌측 패널에만 표시한다.
- 우측 작업 영역에는 좌측과 동일한 Tool Module 목록을 반복 표시하지 않는다.
- 목록은 등록된 Tool Module 루트 폴더의 실제 폴더 계층을 그대로 반영한다.
- 초기 평면 목록에서도 `등록된 도구`, `검증 실패`, `permission 필요` 필터를 제공할 수 있다.
- 루트 폴더 등록 진입은 우측 빈 상태와 도구 화면 상단 액션으로 제공한다.

### 설정 모드

- 일반
- 브라우저
- 단축키
- 기기/권한
- Secret
- 동기화
- 실행 이벤트
- 데이터 관리

## 4. 실행 페이지

실행 페이지는 Workflow 런처이다.

### 표시 방식

그리드와 목록을 제공한다. 기본값은 그리드이다.

그리드 버튼:

- Workflow 이름만 텍스트로 표시한다.
- 버튼 자체가 실행 액션이다.
- 카드와 버튼을 별도 구성으로 나누지 않는다.
- 상태는 텍스트나 배지를 추가하지 않고 버튼 색상과 상태 스타일로 표현한다.
- 기본 배치는 5열이다.
- 그리드 열 수 조정 옵션은 보기 모드 전환 토글 옆에 배치한다.

목록 행:

- Workflow 이름
- 상태
- Action 개수
- 마지막 실행 시간
- 다음 실행 예정 시간
- 마지막 결과 요약
- 실행/중지 버튼

### 동작 규칙

- 실행 페이지에는 Action을 독립 런처로 표시하지 않는다.
- 그리드 모드에서는 각 Workflow 버튼의 클릭이 곧 실행이다.
- 그리드 모드에서는 실행 버튼 외 보조 정보 노출을 최소화한다.
- 그리드 모드에서 실행 중인 Workflow는 버튼 색상과 비활성 상태로만 표현한다.
- 목록 모드와 상세 화면에서는 중지 가능 상태와 중지 액션을 표시한다.
- 실패한 Workflow는 그리드 모드에서는 버튼 색상으로만 표현하고, 목록 모드와 상세 화면에서는 마지막 오류를 요약한다.
- 빈 상태는 `Workflow 작성` 진입을 제공한다.

## 5. Action 편집 화면

Action 편집 화면은 Action 생성과 수정을 같은 화면 구조에서 처리한다.

### 레이아웃

- 좌측 패널: Action 목록 또는 계층 목록
- 좌측 패널 우상단: `+` 아이콘 버튼
- 우측 작업 영역: 선택된 Action 수정 화면
- 우측 작업 영역 빈 상태: 새로 만들기 아이콘과 생성 안내

선택된 Action이 없을 때는 우측에 큰 `+` 아이콘 버튼과 새 Action 생성 안내만 표시한다. 좌측 패널의 `+`와 우측 빈 상태의 `+`는 모두 같은 생성 흐름을 연다.

### Action 목록

표시 항목:

- 이름
- 타입
- 사용 중인 Workflow 수
- secret 필요 여부
- 마지막 수정 시간
- 실행 가능 여부

목록 구조:

- 기본 구현은 평면 목록이다.
- 최종 목표는 폴더 또는 그룹 기반 계층 목록이다.
- 계층 구조가 추가되어도 Action 선택, 생성, 삭제, 필터 동작이 유지되어야 한다.

### Action 생성

생성 폼은 타입 선택 후 타입별 설정으로 전환한다.

공통 필드:

- 이름
- 설명
- 타입
- secret 참조
- 입력/출력 요약

타입별 필드:

- 브라우저: 브라우저 종류, 실행 방식, URL 목록, 동적 업데이트 여부
- Crawler: URL 목록, timeout, 결과 저장 정책
- Discord/Notion dry-run: dry-run 대상, secret 연결 슬롯, 출력 artifact 이름
- Trading dry-run: 전략 이름, 입력값, 실거래 제외 확인
- Tool Action: Tool Module 선택, manifest input 기본값, permission 확인

### 삭제와 영향 범위

- 사용 중인 Action 삭제 시 영향받는 Workflow 목록을 먼저 보여준다.
- 사용 중인 Workflow가 있으면 삭제를 막고 Workflow에서 제거하도록 유도한다.

## 6. Workflow 편집 화면

Workflow 편집 화면은 Workflow 생성과 수정을 같은 화면 구조에서 처리한다. 기본 구성은 Action 편집 화면과 유사하지만, 좌측 목록과 우측 편집 대상이 Workflow이다.

### 레이아웃

- 좌측 패널: Workflow 목록 또는 계층 목록
- 좌측 패널 우상단: `+` 아이콘 버튼
- 우측 작업 영역 빈 상태: 새로 만들기 아이콘과 생성 안내
- 우측 작업 영역 상단: 선택된 Workflow 이름, 설명, 상태
- 우측 작업 영역 본문: 선택된 Workflow의 Action 순서 목록
- 우측 작업 영역 하단 또는 탭: 권한, 예약, 실행 기록, 출력

선택된 Workflow가 없을 때는 우측에 큰 `+` 아이콘 버튼과 새 Workflow 생성 안내만 표시한다. 좌측 패널의 `+`와 우측 빈 상태의 `+`는 모두 같은 생성 흐름을 연다.

### Workflow 목록

- 기본 구현은 평면 목록이다.
- 최종 목표는 폴더 또는 그룹 기반 계층 목록이다.
- 계층 구조가 추가되어도 Workflow 선택, 생성, 삭제, 필터 동작이 유지되어야 한다.

### Action 순서 목록

각 Action row는 다음을 표시한다.

- 순서 번호
- Action 이름과 타입
- enabled 토글
- 입력 연결 상태
- secret 필요 배지
- 마지막 실행 결과
- 이동 버튼
- 제거 버튼

### 입력 연결

- 이전 Action의 output을 이후 Action input에 연결할 수 있다.
- 연결할 수 없는 타입은 비활성화하고 이유를 표시한다.
- 연결이 없는 필수 input은 저장 전 오류로 표시한다.

### 예약과 권한

- 예약 방식: 없음, interval, daily, weekly
- 다음 실행 예정 시간을 저장 전 미리 보여준다.
- visibility/execution policy는 Workflow 단위로 설정한다.
- Action 내부 secret 참조는 Workflow 권한 요약에 함께 표시한다.

## 7. Workflow 상세 화면

Workflow 상세는 실행 후 상태 확인과 관리 중심 화면이다.

탭 구성:

- 개요: 상태, 마지막 실행, 다음 실행, Action 개수, 마지막 메시지
- Action: Action별 상태, 입력 연결, 출력 요약
- 실행 기록: Workflow 이벤트와 Action별 이벤트
- 출력: 마지막 artifact 경로, 결과 요약
- 권한: 기기 정책, secret 참조, 제한 사유
- 예약: schedule 설정과 다음 실행 시각

실행 중에는 Action 순서 변경, Action 제거, 예약 변경을 제한한다. 이름과 설명 같은 메타데이터는 수정할 수 있다.

## 8. 도구 페이지

도구 페이지는 Tool Module 전용이다.

### 레이아웃

- 좌측 패널: 등록된 Tool Module 루트 폴더의 계층 목록
- 우측 작업 영역: 선택된 Tool Module 상세, 검증 결과, 단독 실행, Workflow 추가 액션
- 우측 작업 영역 빈 상태: Tool Module 루트 폴더 등록 진입과 안내

도구 목록은 좌측 패널에만 표시한다. 우측 작업 영역은 선택된 Tool Module의 상세와 실행 UI만 담당하며, 좌측과 동일한 목록을 중복 표시하지 않는다.

### 루트 폴더 등록

도구 등록은 개별 Tool Module 폴더를 하나씩 선택하는 방식이 아니다. 사용자는 Tool Module들을 저장할 루트 폴더 하나를 등록한다. Pastel Flow는 해당 루트 폴더 아래의 폴더 계층을 스캔하고, `tool-schema.md` 규격을 만족하는 모듈을 Tool Module로 인식한다.

동작 규칙:

- 루트 폴더 내부의 폴더 계층은 좌측 패널의 계층 목록에 그대로 반영한다.
- `manifest.json`과 `logic.js`가 있는 폴더를 Tool Module 후보로 본다.
- 규격을 만족하는 후보는 등록된 도구로 표시한다.
- 규격을 만족하지 않는 후보는 검증 실패 상태로 표시하거나, 후보가 아닌 일반 폴더로 취급한다.
- 루트 폴더 변경 시 기존 도구 목록을 새 루트 폴더 기준으로 다시 스캔한다.

검증 결과:

- `manifest.json` 존재 여부
- `logic.js` 존재 여부
- 필수 필드 유효성
- input/output 타입 지원 여부
- permission 선언 유효성
- `view.html`, `style.css`, `README.md` 선택 파일 감지

### 등록된 도구 목록

등록된 도구 목록은 좌측 패널에서만 표시한다.

표시 항목:

- 도구 이름
- id
- version
- description
- permission 목록
- input/output 요약
- 등록 상태

### 단독 실행

- manifest inputs를 기반으로 자동 폼을 만든다.
- 실행 전 permission을 표시한다.
- 실행 결과는 outputs 정의에 맞춰 표시한다.
- 오류는 도구 실행 섹션 안에 표시한다.

### Workflow 추가

- 등록된 도구에서 `Workflow에 추가` 액션을 제공한다.
- 선택 시 Tool Action 생성 화면으로 이동하고 해당 도구를 미리 선택한다.

도구 페이지에는 sync, 실행 이벤트 정리, 로그 보존, 데이터 위치 기능을 두지 않는다.

## 9. 설정 페이지

설정 페이지는 앱 전역 설정과 운영 기능을 다룬다.

### 일반

- 테마: 시스템, 라이트, 다크
- 기본 브라우저
- 새 Action 기본 이름
- 새 Workflow 기본 이름
- 기본 목록 표시 방식

### 브라우저

- Chrome, Edge, Chromium 실행 파일 경로
- 기본 브라우저 실행 정책
- 기존 프로필 경로 사용 경고 정책
- 실행 파일 경로 입력에는 OS별 경로 형식 예시를 표시한다. 예: `C:\Program Files\Google\Chrome\Application\chrome.exe`

## 10. 테마

테마는 반복 사용에 피로가 적은 작업 앱 방향으로 설계한다.

### 테마 모드

- 시스템: OS의 `prefers-color-scheme`을 따른다.
- 라이트: 기본 테마이다.
- 다크: 장시간 실행 모니터링을 위해 제공한다.

### 색상 역할

- 배경
- 표면
- 경계선
- 본문 텍스트
- 보조 텍스트
- 강조
- 위험
- 경고
- 성공

### 구현 규칙

- 색상은 CSS 변수로 정의한다.
- 컴포넌트는 역할 기반 변수만 사용한다.
- 테마 전환은 `data-theme` 속성 또는 루트 클래스 기준으로 적용한다.
- 텍스트 크기와 spacing은 테마에 따라 바꾸지 않는다.

## 11. 공통 컴포넌트 규칙

### 버튼

- 주요 실행: 채워진 버튼
- 보조 액션: 테두리 버튼
- 위험 액션: 붉은 채워진 버튼
- 상단 카테고리 바는 아이콘 중심 버튼을 우선하고 접근 가능한 라벨을 제공한다.
- 실행, 중지, 새로고침, 추가, 삭제, 루트 폴더 등록, 검증에는 아이콘을 우선 사용한다.

### 패널

- 좌측 패널과 우측 작업 영역은 경계선으로만 구분한다.
- 패널 안에 장식용 카드를 중첩하지 않는다.
- 섹션 구분은 얇은 경계선과 작은 제목으로 처리한다.

### 폼

- 라벨은 항상 입력 위에 둔다.
- 필수 입력 실패는 해당 필드 가까이에 표시한다.
- 읽기 전용 값은 입력처럼 보이되 수정 불가 상태를 분명히 한다.
- JSON 또는 배열 입력은 textarea만 제공하지 말고 구조화된 항목 추가/삭제 UI를 우선한다.

### 상태 배지

- 대기: 중립
- 실행 중: 경고
- 실패: 위험
- 성공: 낮은 채도의 긍정 색상
- 제한됨: 중립 또는 경고 색상
- secret 필요: 경고 색상

### 스크롤

- 전체 창은 스크롤하지 않는다.
- 전체 창에 별도의 가로 스크롤바 또는 세로 스크롤바가 생기면 안 된다.
- 좌측 패널과 우측 작업 영역만 각각 독립 세로 스크롤을 가진다.
- 좌측 패널과 우측 작업 영역에는 가로 스크롤을 만들지 않고, 긴 텍스트는 줄바꿈, 말줄임, 또는 내부 코드 블록 처리로 해결한다.
- 스크롤바는 얇고 낮은 대비로 표시한다.

### 좌측 패널 토글

- 토글 버튼은 좌측 패널의 열림/닫힘 상태와 무관하게 앱 프레임의 좌측 끝에 고정한다.
- 패널이 닫혀도 토글 버튼은 같은 x 좌표를 유지한다.
- 패널이 열릴 때 토글 버튼이 패널 폭만큼 오른쪽으로 밀려나지 않아야 한다.
- 토글 버튼은 항상 키보드 포커스와 접근 가능한 라벨을 제공한다.

## 12. 세부 화면 흐름

### 초기 실행

- 앱 시작 시 실행 페이지를 연다.
- 실행 페이지에는 Workflow만 표시한다.
- Workflow가 없으면 Workflow 작성 진입과 Action 생성 진입을 함께 제공한다.

### Action 생성

- 상단 바의 `Action`으로 이동한다.
- 좌측 패널 우상단의 `+` 또는 우측 빈 상태의 새로 만들기 아이콘을 누르면 타입 선택을 먼저 보여준다.
- 타입 선택 후 타입별 폼을 표시한다.
- 저장 성공 후 Action 상세를 유지하고, `Workflow에 추가` 액션을 제공한다.

### Workflow 작성

- 상단 바의 `Workflow`로 이동한다.
- 좌측 패널 우상단의 `+` 또는 우측 빈 상태의 새로 만들기 아이콘을 누르면 이름과 Action 추가 영역을 표시한다.
- Action이 없으면 Action 생성 진입을 제공한다.
- 저장 성공 후 실행 페이지로 돌아갈지, 상세에 남을지는 사용자 설정으로 둔다. 기본값은 상세 유지이다.

### Tool Module 업로드

- 상단 바의 `도구`로 이동한다.
- Tool Module 루트 폴더를 등록한다.
- 등록 후 루트 폴더를 스캔하고 좌측 패널에 계층 목록을 표시한다.
- 모듈 선택 시 우측 작업 영역에 검증 결과, 단독 실행, Workflow 추가 액션을 제공한다.

### 설정 열기

- 상단 바의 `설정`으로 이동한다.
- 좌측 패널에서 설정 카테고리를 선택한다.
- 변경 사항이 있는 상태에서 다른 모드로 이동하면 확인을 표시한다.

## 13. 완료 기준

- 앱 시작 시 실행 페이지에는 Workflow 런처만 보인다.
- Action 관리와 Workflow 작성이 별도 화면으로 분리된다.
- Action과 Workflow 화면은 좌측 계층 확장 가능 목록, 좌측 우상단 `+`, 우측 빈 상태 생성 진입, 선택 대상 수정 화면 구조를 공유한다.
- 실행 페이지 그리드 모드는 Workflow 이름이 적힌 실행 버튼만 5열 기본 배치로 표시한다.
- Workflow 작성 화면은 Action 추가, 순서 조정, enabled 토글, 입력 연결, 권한, 예약을 지원한다.
- 도구 페이지는 Tool Module 업로드, 검증, 관리, 단독 실행, Workflow 추가만 담당한다.
- 도구 페이지는 개별 모듈 폴더가 아니라 Tool Module 루트 폴더를 등록한다.
- 도구 페이지 좌측 패널은 루트 폴더 내부 계층 구조를 따라 Tool Module 목록을 표시한다.
- 도구 목록은 좌측 패널에만 표시되고 우측 작업 영역에는 중복 표시되지 않는다.
- 동기화, 실행 이벤트, 데이터 관리는 설정 페이지에서 접근한다.
- 설정 페이지는 단축키 사용자 정의 카테고리를 제공한다.
- 경로 입력 항목은 사용자가 따라 쓸 수 있는 형식 예시를 함께 제공한다.
- Tool Module 실행 전 permission과 input/output 정의가 표시된다.
- 실행 실패는 Workflow 수준과 Action 수준으로 구분해 표시된다.
- 테마와 공통 컴포넌트 규칙은 모든 화면에 일관되게 적용된다.
- 전체 창 스크롤바는 사용하지 않으며 좌측 패널과 우측 작업 영역의 세로 스크롤만 사용한다.
- 좌측 패널 토글 버튼은 패널 상태와 무관하게 좌측 끝 고정 위치를 유지한다.
