# Pastel Flow 세부 UI 설계

이 문서는 `ui-features.md`의 MVP 기능 범위를 유지하면서 앱 설정, 상단 패널, 테마, 공통 UI 규칙을 더 구체화한다. 구현 우선순위는 저장된 브라우저 작업을 빠르게 실행하는 흐름이며, 설정과 테마는 사용자의 작업 환경을 안정적으로 유지하기 위한 보조 기능으로 둔다.

## 1. 전체 앱 프레임

Pastel Flow의 기본 UI는 창 전체를 쓰는 작업 앱 프레임이다. 채팅 앱처럼 상단 카테고리 바, 좌측 탐색 패널, 우측 작업 영역이 고정된 구조를 기본으로 한다.

### 구성

- 상단 카테고리 바: 앱 이름, 현재 모드, 실행, 새 작업, 수정, 도구, 설정 화면 진입, 새로고침
- 좌측 탐색 패널: 현재 모드에 맞는 보조 탐색, 작업 카테고리, 작업 목록, 설정 분류 표시
- 우측 작업 영역: 현재 모드의 목록, 폼, 상세 정보

### 배치 규칙

- 데스크톱에서는 상단 바 아래에 좌측/우측 2열을 유지한다.
- 좌측 패널은 화면 높이를 채우며 내부 스크롤만 가진다.
- 우측 작업 영역은 목록이나 폼이 길어질 때 독립적으로 스크롤한다.
- 주요 UI는 창 가장자리까지 이어지는 앱 프레임으로 보여야 하며, 전체 화면을 감싼 큰 카드처럼 보이면 안 된다.
- 좁은 화면에서도 동일한 화면 구조를 유지하며, 좌측 패널 토글을 활용할 수 있도록 한다.

## 2. 상단 카테고리 바

상단 카테고리 바는 현재 앱의 모드를 전환한다. 좌측부에는 앱 이름과 정보, 우측부에는 전역 새로고침 버튼을 포함한다.

### 구성

- 좌측: 앱 이름 `Pastel Flow`, 현재 모드 라벨
- 중앙: 실행, 새 작업, 수정, 도구, 설정 토글
- 우측: 새로고침

### 액션

- '실행': 우측 작업 영역을 런처 목록으로 되돌린다.
- '새 작업': 우측 작업 영역에 생성 폼을 단독으로 로딩한다.
- '수정': 수정 화면을 로딩한다.
- '도구': 앱에 내장된 기본 응용 도구들을 사용하는 화면을 로딩한다. 포함할 도구 범위는 아직 정하지 않았으므로 MVP에서는 페이지 구현만 준비한다.
- '설정': 앱 전역 설정 화면을 로딩한다.
- `새로고침`: 로컬 저장소에서 작업 목록을 다시 불러온다.


### 상태 표시

- 저장소 로딩 중이면 새로고침 버튼을 비활성화하고 로딩 상태를 표시한다.
- 공통 오류가 있으면 상단 패널 아래 오류 배너에 표시한다.
- 추후 기기 정책이 들어오면 상단 패널 우측에 현재 기기 신뢰 상태를 짧게 표시한다.

## 3. 좌측 탐색 패널

좌측 패널은 고정된 메뉴가 아니라 현재 모드에 따라 역할과 내용이 바뀌는 보조 패널이다. 실행 모드에서는 작업 목록 필터를 제공하고, 수정 모드에서는 수정할 작업 리스트를 제공하며, 설정 모드에서는 설정 분류를 제공한다.

### 구성


- 패널 토글
- 현재 모드에 맞는 보조 탐색 항목
- 로컬 우선/전용 프로필 같은 짧은 상태 힌트

### 동작 규칙

- `전체`: 표시 가능한 모든 브라우저 작업을 보여준다.
- `폴더`: 현재 MVP에서는 전체 목록과 동일하게 동작하되, 이후 폴더 모델이 추가되면 폴더 범위로 필터링한다.
- `즐겨찾기`: 현재 MVP에서는 빈 범위를 보여주며, 이후 즐겨찾기 모델이 추가되면 즐겨찾기 작업만 보여준다.
- '실행 중': 상태가 `running`인 작업만 실제로 필터링한다.
- `제한됨`: 기기 정책이나 secret 참조가 있는 작업만 보여준다.
- 수정 모드에서는 좌측 패널에 수정할 작업 리스트를 표시하고, 선택된 작업을 우측 작업 영역의 수정 대상으로 사용한다.
- 설정 모드에서는 좌측 패널에 설정 분류를 표시하고, 선택된 분류의 세부 설정을 우측 작업 영역에 표시한다.
- 좌측 패널 토글은 좌측 패널 전체를 접거나 다시 여는 기능이다. 소형 화면에서만 제공할지, 모든 화면 크기에서 제공할지는 추후 결정한다.

## 4. 기본 실행 화면

기본 실행 화면은 저장된 작업 템플릿의 실행 옵션만 보여주는 런처다. 사용자가 앱을 열었을 때 가장 먼저 보는 화면이며, 생성/수정/설정 UI는 이 화면에 섞지 않는다. 작업 목록 표시 설정에 따라 그리드 형식, 목록 형식으로 옵션을 표시한다. 기본은 그리드이다.

### 표시 항목

공통:
- 작업 표시 설정 토글
- 작업명
- 실행 상태
- 실행 버튼

목록 형식 한정:
- 브라우저 종류
- 마지막 실행 시간

### 동작 규칙

- 실행 버튼은 목록에서 바로 누를 수 있어야 한다.
- 실행 중인 작업의 실행 버튼은 비활성화한다.
- 저장된 작업이 없으면 빈 상태와 새 작업 토글 진입을 보여준다.
- 기본 그리드 형식에서는 작업 선택을 제공하지 않고 바로 실행에 집중한다.
- 목록 형식에서 추가 작업을 선택해 수행할 수 있게 할지는 추후 결정한다.
- 목록 표시 형식 토글의 초기값은 그리드이며, 사용자가 우측 작업 영역에서 토글한 값은 전역 설정으로 저장한다.

## 5. 앱 설정

앱 설정은 전역 동작을 조정하는 보조 화면이다. 좌측 패널을 통해 설정의 기본적인 분류를 제공하고, 우측 작업 패널에서 세부 조정 항목들을 표시한다.

### MVP 설정 항목

- 테마: 시스템 설정, 라이트, 다크
- 기본 브라우저: Chrome, Edge, Chromium
- 새 작업 기본 이름: 기본값 `새 브라우저 작업`
- 초기 URL 입력 방식: 줄 단위 입력 유지
- 데이터 위치 보기: Electron `userData` 경로를 읽기 전용으로 표시

### 이후 확장 항목

- 기기 신뢰 상태
- secret 저장소 상태
- 동기화 계정
- adapter 활성화 여부
- 실행 로그 보존 기간

### 동작 규칙

- 설정 변경은 저장 버튼으로 확정한다.
- 저장하지 않고 닫으려 하면 변경 사항이 있는 경우 확인 메시지를 표시한다.
- 저장 성공 후에는 설정 패널을 닫지 않고 성공 상태만 짧게 표시한다.
- 저장 실패 시 패널 내부에 오류를 표시하고 기존 설정값은 유지한다.

### MVP 저장 방식

- 설정 모델이 추가되기 전까지는 UI 설계만 문서화한다.
- 구현 시에는 `appSettings.json` 같은 별도 로컬 설정 파일을 두고 task template 저장소와 분리한다.
- 작업 템플릿의 `config`에는 앱 전역 설정을 저장하지 않는다.

## 6. 테마

테마는 작업 관리 앱답게 차분하고 반복 사용에 피로가 적은 방향으로 설계한다.

### 테마 모드

- 시스템 설정: OS의 `prefers-color-scheme`을 따른다.
- 라이트: 기본 MVP 테마로 사용한다.
- 다크: 야간 사용과 장시간 실행 모니터링을 위해 제공한다.
- 커스텀: 테마의 색상을 각각 사용자 지정한다.(구현 예정)

### 색상 역할

- 배경: 앱 전체 바탕
- 표면: 작업 목록, 상세 패널, 생성 패널
- 경계선: 패널과 행 구분
- 본문 텍스트: 주요 정보
- 보조 텍스트: 메타 정보, 설명, 시간
- 강조: 실행, 선택 상태, 주요 액션
- 위험: 삭제, 실패 상태, 마지막 오류
- 경고: 실행 중, 주의가 필요한 상태

### 라이트 테마 방향

- 전체 배경은 옅은 회색 계열로 둔다.
- 표면은 흰색에 가깝게 유지한다.
- 강조색은 현재 UI의 녹청 계열을 유지하되 버튼과 선택 상태에만 제한적으로 사용한다.
- 실패와 삭제는 붉은 계열을 사용하되, 일반 정보와 섞이지 않게 분리한다.

### 다크 테마 방향

- 전체 배경은 푸른색 계열 검정 색상을 사용한다.
- 표면은 배경보다 한 단계 밝게 둔다.
- 경계선은 낮은 대비로 사용해 패널 구분만 만든다.
- 강조색은 라이트 테마보다 채도를 낮추고, 텍스트 대비를 우선한다.

### 구현 규칙

- 색상은 CSS 변수로 정의한다.
- 컴포넌트는 직접 색상값보다 역할 기반 변수만 사용한다.
- 테마 전환은 `data-theme` 속성 또는 루트 클래스 기준으로 적용한다.
- 텍스트 크기와 spacing은 테마에 따라 바꾸지 않는다.

## 7. 공통 컴포넌트 규칙

### 버튼

- 주요 실행: 채워진 버튼
- 보조 액션: 테두리 버튼
- 위험 액션: 붉은 채워진 버튼
- 상단 카테고리 바는 텍스트 없이 깔끔한 이미지 아이콘만 사용하는 세그먼트 버튼을 권장한다. 각 버튼에는 접근 가능한 라벨을 제공한다.
- 좌측 탐색은 아이콘과 짧은 라벨을 함께 쓴다.
- 새로고침, 실행, 새 작업, 수정, 설정, 닫기, 삭제에는 아이콘을 우선 사용하고 접근 가능한 라벨을 제공한다.

### 패널

- 좌측 패널, 우측 작업 패널은 같은 색상 역할을 공유하되 서로 카드처럼 떠 보이지 않게 경계선으로만 구분한다.
- 생성 패널, 상세 패널, 설정 패널은 우측 작업 영역 안에서 같은 시각 규칙을 공유한다.
- 패널 안에 다시 장식용 카드를 중첩하지 않는다.
- 섹션 구분은 얇은 경계선과 작은 제목으로 처리한다.

### 폼

- 라벨은 항상 입력 위에 둔다.
- 읽기 전용 값은 입력처럼 보이되 색을 낮춰 수정 불가 상태를 분명히 한다.
- 필수 입력 실패는 해당 필드 가까이에 표시한다.

### 상태 배지

- 대기: 중립 색상
- 실행 중: 경고 색상
- 실패: 위험 색상
- 성공 상태가 필요해지면 강조색보다 낮은 채도의 긍정 색상을 사용한다.

### 스크롤

- 브라우저 기본 스크롤바가 강하게 드러나지 않도록 얇은 스크롤바를 사용한다.
- 전체 창은 기본적으로 스크롤하지 않고, 좌측 패널과 우측 작업 영역이 필요할 때 각자 스크롤한다.
- 스크롤바 트랙은 투명하게 두고 thumb만 낮은 대비로 표시한다.

## 8. 세부 화면 흐름

### 초기 실행

- 앱 시작 시 실행 화면을 연다.
- 실행 화면에는 작업 템플릿 실행 옵션만 표시한다.
- 지나치게 작은 세로가 긴 소형 화면의 경우 좌측 패널의 토글 기본값을 꺼짐으로 한다.(설정 변경 가능)

### 새 작업 생성

- 상단 바의 `새 작업`으로 생성 화면을 로딩한다.
- 기본 브라우저는 앱 설정의 기본값을 사용한다.
- 실행 방식은 `전용 프로필` 읽기 전용으로 표시한다.
- 생성 성공 시 자동으로 실행 화면으로 돌아간다(설정 변경 가능).

### 기존 작업 수정

- 상단 바의 `수정`으로 수정 화면을 로딩한다.
- 새롭게 로딩된 좌측 패널에서 선택한 작업을 수정 대상으로 사용한다.
- 저장 성공 후에는 수정 화면을 유지하고 성공 상태를 짧게 표시한다.
- 삭제 성공 후에는 다음 작업을 선택하고, 남은 작업이 없으면 실행 화면의 빈 상태로 돌아간다.

### 앱 설정 열기

- 상단 바의 `설정`으로 설정 화면을 로딩한다.
- 설정 화면은 좌측 패널과 우측 작업 영역 모두 설정 전용 UI로 재정의한다. 좌측 패널에는 설정 분류를 표시하고, 우측 작업 영역에는 선택된 분류의 세부 설정을 표시한다.
- 설정 화면을 닫거나 버튼을 다시 클릭, 혹은 좌측 탐색의 실행을 누르면 기본 실행 화면으로 돌아간다.

### 테마 변경

- 설정 패널에서 테마를 선택한다.
- 저장 전에는 설정 화면 내에서 테마를 반영한다.
- 저장 후 앱 전체 테마에 반영한다.
- 저장하지 않고 나갈 경우 테마는 복원된다.
- 시스템 설정을 선택하면 OS 테마 변경을 따라간다.

## 9. 완료 기준

- 앱 시작 시 기본 실행 화면에는 작업 템플릿 실행 옵션만 보인다.
- 창 전체가 상단 카테고리 바, 좌측 탐색 패널, 우측 작업 영역으로 분리된다.
- 상단 카테고리 바는 작업 모드 전환과 새로고침만 담당한다.
