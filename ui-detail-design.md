# Pastel Flow 세부 UI 설계

이 문서는 `ui-features.md`의 MVP 기능 범위를 유지하면서 앱 설정, 상단 패널, 테마, 공통 UI 규칙을 더 구체화한다. 구현 우선순위는 저장된 브라우저 작업을 빠르게 실행하는 흐름이며, 설정과 테마는 사용자의 작업 환경을 안정적으로 유지하기 위한 보조 기능으로 둔다.

## 1. 상단 패널

상단 패널은 앱의 현재 모드와 가장 자주 쓰는 전역 액션을 제공한다. 초기 진입 모드는 `실행`이며, 사용자는 상단 아이콘 토글로만 `새 작업`, `수정`, `설정` 화면을 로딩한다.

### 구성

- 좌측: 앱 이름 `Pastel Flow`, 현재 작업 영역 설명
- 중앙: 현재 모드 라벨 또는 이후 검색/필터 영역
- 우측: 새로고침, 실행, 새 작업, 수정, 앱 설정 아이콘 토글

### 액션

- `새로고침`: 로컬 저장소에서 작업 목록을 다시 불러온다.
- `실행`: 초기 런처 화면으로 돌아간다.
- `새 작업`: 브라우저 작업 생성 화면을 로딩한다.
- `수정`: 선택한 브라우저 작업 수정 화면을 로딩한다.
- `설정`: 앱 설정 화면을 로딩한다.

### 상태 표시

- 저장소 로딩 중이면 새로고침 버튼을 비활성화하고 로딩 상태를 표시한다.
- 공통 오류가 있으면 상단 패널 아래 오류 배너에 표시한다.
- 추후 기기 정책이 들어오면 상단 패널 우측에 현재 기기 신뢰 상태를 짧게 표시한다.

### 배치 규칙

- 데스크톱에서는 한 줄 레이아웃을 유지한다.
- 좁은 화면에서는 앱 이름이 위, 액션 버튼이 아래로 내려간다.
- 상단 패널은 화면을 과도하게 차지하지 않고 실행 화면의 시작점을 빠르게 보여준다.

## 2. 기본 실행 화면

기본 실행 화면은 저장된 작업 템플릿의 실행 옵션만 보여주는 런처다. 사용자가 앱을 열었을 때 가장 먼저 보는 화면이며, 생성/수정/설정 UI는 이 화면에 섞지 않는다.

### 표시 항목

- 작업명
- 브라우저 종류
- 실행 상태
- 마지막 실행 시간
- 실행 버튼

### 동작 규칙

- 실행 버튼은 목록에서 바로 누를 수 있어야 한다.
- 실행 중인 작업의 실행 버튼은 비활성화한다.
- 작업 행 선택은 이후 수정 화면에서 사용할 작업을 고르는 의미만 가진다.
- 작업 행 선택만으로 상세 패널, 수정 폼, 삭제 UI가 열리지 않는다.
- 저장된 작업이 없으면 빈 상태와 새 작업 토글 진입을 보여준다.

## 3. 앱 설정

앱 설정은 전역 동작을 조정하는 보조 화면이다. MVP에서는 복잡한 설정 라우트보다 상단 아이콘 토글로 현재 작업 영역에 로딩되는 단일 패널을 사용한다.

### MVP 설정 항목

- 테마: 시스템 설정, 라이트, 다크
- 기본 브라우저: Chrome, Edge, Chromium
- 새 작업 기본 이름: 기본값 `새 브라우저 작업`
- 초기 URL 입력 방식: 줄 단위 입력 유지
- 데이터 위치 보기: Electron `userData` 경로를 읽기 전용으로 표시

### 이후 확장 항목

- 기기 신뢰 상태
- secret 저장소 상태
- 동기화 계정
- adapter 활성화 여부
- 실행 로그 보존 기간

### 동작 규칙

- 설정 변경은 저장 버튼으로 확정한다.
- 저장하지 않고 닫으려 하면 변경 사항이 있는 경우 확인 메시지를 표시한다.
- 저장 성공 후에는 설정 패널을 닫지 않고 성공 상태만 짧게 표시한다.
- 저장 실패 시 패널 내부에 오류를 표시하고 기존 설정값은 유지한다.

### MVP 저장 방식

- 설정 모델이 추가되기 전까지는 UI 설계만 문서화한다.
- 구현 시에는 `appSettings.json` 같은 별도 로컬 설정 파일을 두고 task template 저장소와 분리한다.
- 작업 템플릿의 `config`에는 앱 전역 설정을 저장하지 않는다.

## 4. 테마

테마는 작업 관리 앱답게 차분하고 반복 사용에 피로가 적은 방향으로 설계한다.

### 테마 모드

- 시스템 설정: OS의 `prefers-color-scheme`을 따른다.
- 라이트: 기본 MVP 테마로 사용한다.
- 다크: 야간 사용과 장시간 실행 모니터링을 위해 제공한다.

### 색상 역할

- 배경: 앱 전체 바탕
- 표면: 작업 목록, 상세 패널, 생성 패널
- 경계선: 패널과 행 구분
- 본문 텍스트: 주요 정보
- 보조 텍스트: 메타 정보, 설명, 시간
- 강조: 실행, 선택 상태, 주요 액션
- 위험: 삭제, 실패 상태, 마지막 오류
- 경고: 실행 중, 주의가 필요한 상태

### 라이트 테마 방향

- 전체 배경은 옅은 회색 계열로 둔다.
- 표면은 흰색에 가깝게 유지한다.
- 강조색은 현재 UI의 녹청 계열을 유지하되 버튼과 선택 상태에만 제한적으로 사용한다.
- 실패와 삭제는 붉은 계열을 사용하되, 일반 정보와 섞이지 않게 분리한다.

### 다크 테마 방향

- 전체 배경은 완전한 검정보다 어두운 회색 계열을 사용한다.
- 표면은 배경보다 한 단계 밝게 둔다.
- 경계선은 낮은 대비로 사용해 패널 구분만 만든다.
- 강조색은 라이트 테마보다 채도를 낮추고, 텍스트 대비를 우선한다.

### 구현 규칙

- 색상은 CSS 변수로 정의한다.
- 컴포넌트는 직접 색상값보다 역할 기반 변수만 사용한다.
- 테마 전환은 `data-theme` 속성 또는 루트 클래스 기준으로 적용한다.
- 텍스트 크기와 spacing은 테마에 따라 바꾸지 않는다.

## 5. 공통 컴포넌트 규칙

### 버튼

- 주요 실행: 채워진 버튼
- 보조 액션: 테두리 버튼
- 위험 액션: 붉은 채워진 버튼
- 상단 전역 액션은 텍스트 버튼보다 아이콘 토글을 우선 사용한다.
- 새로고침, 실행, 새 작업, 수정, 설정, 닫기, 삭제에는 아이콘을 우선 사용하고 접근 가능한 라벨을 제공한다.

### 패널

- 생성 패널, 상세 패널, 설정 패널은 같은 시각 규칙을 공유한다.
- 패널 안에 다시 장식용 카드를 중첩하지 않는다.
- 섹션 구분은 얇은 경계선과 작은 제목으로 처리한다.

### 폼

- 라벨은 항상 입력 위에 둔다.
- 읽기 전용 값은 입력처럼 보이되 색을 낮춰 수정 불가 상태를 분명히 한다.
- 필수 입력 실패는 해당 필드 가까이에 표시한다.

### 상태 배지

- 대기: 중립 색상
- 실행 중: 경고 색상
- 실패: 위험 색상
- 성공 상태가 필요해지면 강조색보다 낮은 채도의 긍정 색상을 사용한다.

## 6. 세부 화면 흐름

### 초기 실행

- 앱 시작 시 실행 화면을 연다.
- 실행 화면에는 작업 템플릿 실행 옵션만 표시한다.
- 수정이나 설정 확인이 필요하면 상단 아이콘 토글을 사용한다.

### 새 작업 생성

- 상단 패널의 `새 작업` 아이콘 토글로 생성 화면을 로딩한다.
- 기본 브라우저는 앱 설정의 기본값을 사용한다.
- 실행 방식은 `전용 프로필` 읽기 전용으로 표시한다.
- 생성 성공 시 새 작업을 자동 선택하고 실행 화면으로 돌아간다.

### 기존 작업 수정

- 상단 패널의 `수정` 아이콘 토글로 수정 화면을 로딩한다.
- 실행 화면에서 마지막으로 선택한 작업을 기본 수정 대상으로 사용한다.
- 수정 화면 안에서 수정할 작업을 바꿀 수 있다.
- 저장 성공 후에는 수정 화면을 유지하고 성공 상태를 짧게 표시한다.
- 삭제 성공 후에는 다음 작업을 선택하고, 남은 작업이 없으면 실행 화면의 빈 상태로 돌아간다.

### 앱 설정 열기

- 상단 패널의 `설정` 아이콘 토글로 설정 화면을 로딩한다.
- 설정 화면은 실행 목록이나 수정 화면과 동시에 노출하지 않는다.
- 설정 화면을 닫거나 실행 토글을 누르면 기본 실행 화면으로 돌아간다.

### 테마 변경

- 설정 패널에서 테마를 선택한다.
- 저장 전에는 패널 내부 미리보기만 바뀐다.
- 저장 후 앱 전체 테마에 반영한다.
- 시스템 설정을 선택하면 OS 테마 변경을 따라간다.

## 7. 완료 기준

- 앱 시작 시 기본 실행 화면에는 작업 템플릿 실행 옵션만 보인다.
- 상단 패널에서 새로고침, 실행, 새 작업, 수정, 설정 아이콘 토글이 명확히 보인다.
- 새 작업 정의, 기존 작업 수정, 설정 화면은 상단 아이콘 토글로만 로딩된다.
- 앱 설정 화면은 테마와 기본 브라우저 설정을 포함한다.
- 설정은 작업 템플릿 데이터와 분리된 전역 설정으로 정의된다.
- 테마는 역할 기반 CSS 변수로 설계된다.
- 라이트/다크/시스템 테마의 동작 기준이 문서화된다.
- MVP 기능 범위인 브라우저 탭 그룹 실행 흐름을 방해하지 않는다.
