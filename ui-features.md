# Pastel Flow MVP UI 기능 명세

이 문서는 Pastel Flow MVP에서 브라우저 탭 그룹 화면이 제공해야 하는 기능과 상태를 정의한다. `PLAN.md`는 제품 방향과 로드맵을, `structure.md`는 코드 구조와 책임 경계를 다루며, 이 문서는 사용자가 화면에서 실제로 수행할 수 있는 행동과 완료 기준을 다룬다.

## 1. 범위

MVP UI는 브라우저 탭 그룹 템플릿만 다룬다.

- 사용자는 저장된 브라우저 작업을 빠르게 찾고 실행할 수 있어야 한다.
- 사용자는 새 브라우저 작업을 만들 수 있어야 한다.
- 사용자는 선택한 작업의 설정과 실행 상태를 확인할 수 있어야 한다.
- 사용자는 선택한 작업의 이름, 브라우저, 초기 URL을 수정할 수 있어야 한다.
- 사용자는 선택한 작업을 확인 절차 후 삭제할 수 있어야 한다.
- MVP에서는 `전용 프로필` 실행 방식만 노출한다.

MVP UI에서 다루지 않는 항목:

- 브라우저 확장 프로그램 실행 방식
- 기본 브라우저 연결 실행 방식
- 브라우저 탭 그룹 외 작업 타입
- 기기 권한, secret, 서버 동기화 설정

## 2. 메인 화면 구조

메인 화면은 다음 세 영역으로 구성한다.

- 상단 헤더: 앱 이름, 새로고침, 새 작업 생성 진입
- 작업 목록: 저장된 브라우저 작업의 빠른 실행 영역
- 상세 패널: 선택된 작업의 설정, 상태, 수정, 삭제 영역

작업 생성 폼은 상시 노출하지 않고 `새 작업` 버튼으로 여는 보조 패널로 제공한다.

## 3. 작업 목록

작업 목록은 이미 저장된 템플릿 실행을 최우선으로 한다.

각 작업 항목은 다음 정보를 보여준다.

- 작업명
- 브라우저 종류
- 실행 상태
- 마지막 실행 시간
- 실행 버튼

목록 항목을 선택하면 상세 패널이 해당 작업으로 갱신된다. 실행 버튼은 목록에서 바로 사용할 수 있어야 하며, 실행 중인 작업의 버튼은 비활성화한다.

## 4. 상세 패널

상세 패널은 선택된 작업의 설정과 실행 상태를 보여준다.

표시 항목:

- 이름
- 브라우저
- 실행 방식: 전용 프로필
- 초기 URL 목록
- 프로필 ID
- 로컬 프로필 경로
- 상태
- 마지막 실행 시간
- 마지막 오류
- 생성 시간
- 수정 시간

수정 모드에서 변경 가능한 항목:

- 이름
- 브라우저
- 초기 URL

삭제는 상세 패널에서 제공하며, 삭제 전 확인 단계를 거친다.

## 5. 상태와 오류

빈 상태:

- 저장된 작업이 없으면 빈 상태 문구와 `새 작업` 버튼을 보여준다.

로딩 상태:

- 작업 목록을 불러오는 동안 목록 영역과 상세 패널에 로딩 상태를 표시한다.

실행 중 상태:

- 실행 중인 작업은 목록과 상세 패널에서 실행 버튼을 비활성화한다.

실행 성공:

- `status`, `lastRunAt`, `localProfilePath`를 화면에 반영한다.

실행 실패:

- 목록에는 실패 상태를 요약한다.
- 상세 패널에는 `lastError`를 사용자가 읽을 수 있게 보여준다.

공통 오류:

- Pastel Flow API 불러오기 실패, 저장소 접근 실패, 생성/수정/삭제 실패는 화면 상단 오류 영역에 표시한다.

## 6. 완료 기준

- 저장된 작업이 없을 때 빈 상태와 생성 진입이 보인다.
- 저장된 작업이 있을 때 목록에서 바로 실행할 수 있다.
- 작업을 선택하면 상세 패널에 설정과 상태가 표시된다.
- 새 작업 생성 후 목록에 추가되고 자동 선택된다.
- 작업 수정 후 목록과 상세 패널의 값이 함께 갱신된다.
- 작업 삭제 전 확인 단계가 나오고, 삭제 후 목록에서 사라진다.
- 실행 성공 시 마지막 실행 시간, 로컬 프로필 경로, 상태가 UI에 반영된다.
- 실행 실패 시 목록 상태와 상세 오류 메시지가 UI에 반영된다.
- MVP UI에서는 전용 프로필 외 실행 방식이 선택지로 보이지 않는다.
