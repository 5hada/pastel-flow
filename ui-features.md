# Pastel Flow MVP UI 기능 명세

이 문서는 Pastel Flow MVP에서 브라우저 탭 그룹 화면이 제공해야 하는 기능과 상태를 정의한다. `PLAN.md`는 제품 방향과 로드맵을, `structure.md`는 코드 구조와 책임 경계를 다루며, 이 문서는 사용자가 화면에서 실제로 수행할 수 있는 행동과 완료 기준을 다룬다.

## 1. 범위

MVP UI는 브라우저 탭 그룹 템플릿만 다룬다.

- 사용자는 저장된 브라우저 작업을 빠르게 찾고 실행할 수 있어야 한다.
- 사용자는 새 브라우저 작업을 만들 수 있어야 한다.
- 사용자는 선택한 작업의 설정과 실행 상태를 확인할 수 있어야 한다.
- 사용자는 선택한 작업의 이름, 브라우저, 초기 URL을 수정할 수 있어야 한다.
- 사용자는 선택한 작업을 확인 절차 후 삭제할 수 있어야 한다.
- MVP에서는 `전용 프로필` 실행 방식만 노출한다.

MVP UI에서 다루지 않는 항목:

- 브라우저 확장 프로그램 실행 방식
- 기본 브라우저 연결 실행 방식
- 브라우저 탭 그룹 외 작업 타입
- 기기 권한, secret, 서버 동기화 설정

## 2. 메인 화면 구조

메인 화면은 창 전체를 사용하는 앱 프레임으로 구성한다. 카드형 모듈을 가운데에 배치하지 않고, 채팅 앱처럼 고정된 상단 카테고리와 좌측의 서브 패널과 우측의 메인 작업 영역으로 양분한다. 앱을 처음 열면 우측 작업 영역은 항상 `실행` 모드로 시작한다.

- 상단 카테고리 바: 앱 이름, 현재 모드, 실행, 새 작업, 수정, 도구, 설정, 새로고침
- 좌측 패널: 패널 토글, 현재 모드에 맞는 보조 탐색 또는 분류
- 우측 작업 영역: 실행 목록, 새 작업 폼, 수정 폼, 설정 폼 중 현재 모드의 주 화면, 목록 표시 형식 토글
- 실행 모드: 저장된 브라우저 작업 템플릿의 실행 옵션만 보여주는 기본 화면
- 새 작업 모드: 브라우저 작업 템플릿을 새로 정의하는 화면
- 수정 모드: 기존 브라우저 작업 템플릿을 선택하고 수정 또는 삭제하는 화면
- 도구 모드: 앱에 내장된 기본 응용 도구들을 사용하는 화면. 포함할 도구 범위는 아직 정하지 않았으므로 MVP에서는 페이지 구현만 준비한다.
- 설정 모드: 앱 전역 설정을 수정하는 화면

좌측 패널은 고정된 단일 메뉴가 아니라 현재 모드에 맞게 내용이 바뀌는 보조 패널이다. 실행 모드에서는 전체/실행 중/폴더/즐겨찾기/제한됨 카테고리로 작업 목록의 범위를 전환하고, 수정 모드에서는 수정할 작업 목록을 제공하며, 설정 모드에서는 설정 분류를 제공한다. 상단 카테고리 바는 앱의 주요 화면을 전환한다. 새 작업 정의, 기존 작업 수정, 도구 열기, 앱 설정 등은 우측 작업 영역에 단독으로 로딩된다.

좌측 패널 토글은 좌측 패널 전체를 접거나 다시 여는 기능이다. 소형 화면에서만 제공할지, 모든 화면 크기에서 제공할지는 추후 결정한다.

## 3. 작업 목록

초기 작업 목록은 이미 저장된 템플릿 실행을 최우선으로 한다. 이 화면은 런처이므로 생성 폼, 수정 폼, 설정 폼, 도구 폼, 삭제 확인 등의 타 UI를 포함하지 않는다. 또한 목록 표시 형식 토글의 상태에 따라 그리드 형식, 목록 형식으로 작업을 표시한다.

각 작업 항목은 표시 형식에 따라 다음 정보를 보여준다.

목록 형식:

- 작업명
- 브라우저 종류
- 실행 상태
- 마지막 실행 시간
- 실행 버튼

그리드 형식:

- 작업명
- 실행 상태
- 실행 버튼

실행 버튼은 목록에서 바로 사용할 수 있어야 하며, 실행 중인 작업의 버튼은 비활성화한다. 실행 모드의 기본 표시 형식인 그리드 형식에서는 작업 선택을 제공하지 않고 바로 실행에 집중한다. 목록 형식에서 추가 작업을 선택해 수행할 수 있게 할지는 추후 결정한다.

좌측 패널의 `실행 중` 카테고리는 상태가 `running`인 작업만 실제로 필터링한다.

목록 표시 형식 토글의 초기값은 그리드 형식이다. 사용자가 우측 작업 영역에서 토글한 값은 전역 설정으로 저장되어 다음 실행에도 유지된다.

## 4. 새 작업과 수정 화면

새 작업 화면은 상단 바의 `새 작업` 액션으로 로딩한다.

입력 항목:

- 이름
- 브라우저
- 실행 방식: 전용 프로필
- 초기 URL 목록

수정 화면은 상단 바의 `수정` 액션으로 로딩한다. 수정 화면에서는 선택된 작업의 설정과 실행 상태를 확인 및 수정할 수 있다.
작업은 좌측 패널에 로딩되는 작업 리스트를 통해 선택할 수 있다.

표시 항목:

- 이름
- 브라우저
- 실행 방식: 전용 프로필
- 초기 URL 목록
- 프로필 ID
- 로컬 프로필 경로
- 상태
- 마지막 실행 시간
- 마지막 오류
- 생성 시간
- 수정 시간

수정 모드에서 변경 가능한 항목:

- 이름
- 브라우저
- 초기 URL

삭제는 수정 화면에서 제공하며, 삭제 전 확인 단계를 거친다. 실행 모드에서는 삭제 UI를 노출하지 않는다.

## 5. 상태와 오류

빈 상태:

- 저장된 작업이 없으면 빈 상태 문구와 `새 작업` 버튼을 보여준다.

로딩 상태:

- 작업 목록을 불러오는 동안 목록 영역과 상세 패널에 로딩 상태를 표시한다.

실행 중 상태:

- 실행 중인 작업은 목록과 상세 패널에서 실행 버튼을 비활성화한다.

실행 성공:

- `status`, `lastRunAt`, `localProfilePath`를 화면에 반영한다.

실행 실패:

- 목록에는 실패 상태를 요약한다.
- 상세 패널에는 `lastError`를 사용자가 읽을 수 있게 보여준다.

공통 오류:

- Pastel Flow API 불러오기 실패, 저장소 접근 실패, 생성/수정/삭제 실패는 화면 상단 오류 영역에 표시한다.

## 6. 완료 기준

- 작업의 기본 표시 형식은 그리드 형식으로 한다.
- 저장된 작업이 없을 때 빈 상태와 생성 진입이 보인다.
- 저장된 작업이 있을 때 목록에서 바로 실행할 수 있다.
- 초기 실행 화면에는 작업 템플릿의 실행 옵션만 표시된다.
- 새 작업, 수정, 도구, 설정 화면은 상단 바로 로딩된다.
- 창 전체가 상단 바, 좌측 패널, 우측 작업 영역으로 나뉘며 카드형 모듈 레이아웃으로 보이지 않는다.
- 작업을 선택한 뒤 수정 토글을 열면 해당 작업의 설정과 상태가 표시된다.
- 새 작업 생성 후 목록에 추가되고 실행 화면으로 돌아오며 자동 선택된다.
- 작업 수정 후 목록과 수정 화면의 값이 함께 갱신된다.
- 작업 삭제 전 확인 단계가 나오고, 삭제 후 목록에서 사라진다.
- 실행 성공 시 마지막 실행 시간, 로컬 프로필 경로, 상태가 UI에 반영된다.
- 실행 실패 시 목록 상태와 상세 오류 메시지가 UI에 반영된다.
- MVP UI에서는 전용 프로필 외 실행 방식이 선택지로 보이지 않는다.
